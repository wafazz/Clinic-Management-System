PHASE 1 · LEAD CAPTURE

Bringing Prospects In

Sales Team

1. Capture

/leads/create
  • Source: FB, walk-in, phone, referral
  • Name, phone, IC, gender
  • Service interest
  • Assign to sales rep

2. Follow-up

/leads/{id}
  • 15 status options:
  • new → contacted
  • followup_1..5
  • appointment_booked / kiv / reject
  • next_followup_at scheduled

3. Convert

/leads/{id}/convert
  • Auto-creates patient record
  • Inherits IC, gender, DOB
  • Generates patient_id (HQ-P00001)
  • Lead marked as success
📊 Stats Dashboard: Total leads · New · Follow-up · Converted — visible at /leads
PHASE 2 · ARRIVAL

Patient Walks Through The Door

Receptionist

Walk-In Path

/walk-in-queue/create
  • Existing patient: select from list
  • New: register first, then queue
  • Number format: W001
  • Auto-priority if member with priority tier

Appointment Path

/walk-in-queue/check-in/{appt}
  • Patient booked online or phoned ahead
  • Check-in by appointment ID
  • Number format: A001
  • Appointment status: confirmed → in-progress

Queue Display + Call Next

/walk-in-queue/display · TV display in waiting area
  • Shows current serving + waiting list
  • ⭐ Priority members bumped to front (red star badge)
  • Receptionist or Doctor presses Call Next → status: waiting → serving
PHASE 3 · CLINICAL ENCOUNTER (1/3)

Vitals → Consultation Start

Nurse Doctor

Vitals (Nurse)

/consultations/{id}/edit
  • Blood Pressure (mmHg)
  • Pulse (bpm)
  • Temperature (°C)
  • Weight + Height → BMI auto-calculated
  • SpO2 (%) · Respiratory Rate

Clinical Notes (Doctor)

/consultations/start
  • Chief complaint
  • History of present illness
  • Physical examination
  • Diagnosis (primary + secondary)
  • Treatment plan + Follow-up date
Consultation Number: CN-{branch}-{YYYYMMDD}-{seq} · e.g. CN-HQ-20260425-0001
PHASE 3 · CLINICAL ENCOUNTER (2/3)

Three Standard Outputs

Doctor can issue any combination of these during the consultation.

Prescription

Doctor /prescriptions/create
  • Multi-item
  • Dosage / frequency / duration
  • Quantity + special instructions
  • Status: draft → dispensed

Lab Order

Doctor → Tech /lab-reports/create
  • Pick from test catalog
  • Lab tech enters results
  • Marks abnormal flags
  • Doctor reviews + finalizes

Medical Cert

Doctor /consultations/{id}/mc-print
  • Inline on consultation form
  • Days from-to (auto count)
  • Reason
  • Printable certificate
PHASE 3 · CLINICAL ENCOUNTER (3/3)

Optional Extensions

Treatment Plan

Doctor /treatment-plans/create
  • Multi-session plan
  • Auto-generates sessions × interval days
  • Mark each session complete
  • Auto-completes when all done

Referral

Doctor /referrals/create
  • To external specialist / hospital
  • Urgency: routine / urgent / emergency
  • Clinical summary + reason
  • Status: pending → sent → completed

Subscription Usage

System (auto) SubscriptionUsage
  • If patient has active subscription
  • +1 visit logged automatically on complete
  • Free consultation tracked vs membership
  • Savings logged to MembershipUsageLog
✓ Complete & Bill — Doctor clicks once, system: marks consultation complete · syncs queue + appointment · auto-redirects to invoice creation
PHASE 4 · DISPENSING & BILLING (1/2)

Dispense + Invoice

Pharmacist Cashier

Dispensing

/prescriptions/{id}/dispense
  • Pharmacist clicks Dispense
  • Stock auto-deducted per item
  • StockMovement audit row written
  • Rx status: draft → dispensed
  • Low-stock notification fires if threshold hit

Invoice Creation

/invoices/create?consultation_id={id}
  • Auto pre-fills:
  • · Consultation fee
  • · Dispensed Rx items
  • · Lab tests
  • Membership banner + auto-discount toggle
💳 Membership Discount — Tier discounts (consultation/medicine/lab %) applied per line item. Logged to membership_usage_logs. Patient's total savings tracked.
PHASE 4 · DISPENSING & BILLING (2/2)

Payment Path

Cash / Card / Online

Cashier /invoices/{id}/payments
  • Multiple payments per invoice OK
  • Status: issued → partial → paid
  • Receipt printable

Insurance Panel

Cashier Admin /insurance-claims/create
  • Submit claim with GL number
  • Workflow: submitted → approved/rejected → paid
  • Annual limit auto-deducted on insurer payment
  • Patient copay tracked separately
📋 Invoice Number Format: {branch}-INV-{YYYYMM}-{seq} · e.g. HQ-INV-202604-0001
PHASE 5 · CLOSE-OUT

Patient Leaves · System Records

Close Queue

Receptionist
  • Status → completed
  • completed_at timestamped
  • Queue cleared from TV

Patient Portal

Patient
  • View invoice
  • View lab reports
  • View prescriptions
  • Self-serve from /portal

Notifications

System (auto)
  • Bell icon dropdown
  • Fires on key events
  • Per-user, in-app

Audit Log

System (auto)
  • Auditable trait
  • Old/new value diff
  • Admin views /audit-logs
✓ Visit Complete — Patient is gone, but the data thread (lead → patient → consultation → Rx → lab → invoice → claim → audit) is fully linked and queryable.
